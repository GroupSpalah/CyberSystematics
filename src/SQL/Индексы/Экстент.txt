Экстент (Extent) в контексте баз данных обозначает непрерывную физическую область на диске, выделенную для хранения данных. Экстент представляет собой логическую группу блоков данных в базе данных, и они обычно используются для эффективного управления пространством на диске.

Экстент отличается от блока данных, который представляет собой минимальную единицу хранения данных в базе данных. Вместо того чтобы хранить данные блок за блоком, база данных может выделять и управлять пространством в виде экстентов. Это может снизить фрагментацию на диске и повысить эффективность управления пространством.

Обычно экстент имеет фиксированный размер, который задается при создании базы данных. Когда база данных нуждается в дополнительном пространстве для хранения данных, ей предоставляется новый экстент.

Существует два основных типа экстентов:

1. **Uniform Extent (Равномерный экстент):** В этом случае все экстенты в базе данных имеют одинаковый размер.

2. **Mixed Extent (Смешанный экстент):** Здесь разные таблицы или индексы могут использовать экстенты разного размера в зависимости от их потребностей. Этот подход обеспечивает более гибкое управление пространством.

Использование экстентов позволяет уменьшить фрагментацию на диске и повысить эффективность использования пространства. Этот механизм управления пространством является частью стратегии управления хранилищем данных в базах данных.
